# Implementation Plan

## Overview

План реализации MVP сервиса сайтов-приглашений на Next.js (App Router, React, TypeScript) + Node.js + PostgreSQL (Prisma). Идёт от инфраструктуры и data-driven движка шаблонов к API, оплате, рантайму сценария, трём шаблонам, уведомлениям в Telegram и кабинету автора. Каждая задача ссылается на требования и, где применимо, на инварианты (Property N) из дизайна.

- [x] 5.1 Абстракция PaymentProvider и checkout

  - Тесты согласованности тарифа (Property 9).
  - _Requirements: 3.5, 3.6_

  - Интеграционные тесты OG-метаданных и noindex (Property 6), недоступной ссылки (Property 7).
  - _Requirements: 4.1, 4.2, 4.4, 11.2, 11.3, 11.4_

  - Юнит-тесты переходов и развилок для всех шаблонов.
  - _Requirements: 5.3, 5.4_

  - _Requirements: 6.2, 6.3_

  - Интеграционные тесты идемпотентности ответа (Property 3) и повторного открытия.
  - _Requirements: 5.5, 5.7, 8.5_

- [x] 8. Экраны шаблонов (UI каждого шаблона)

## Notes

- Тесты включены внутрь соответствующих задач (юнит/интеграционные/компонентные) — отдельной задачи на тесты нет.
- Ручная проверка во встроенных браузерах Telegram/WhatsApp/Instagram обязательна перед запуском (см. Testing Strategy в design.md) и выполняется в рамках задачи 11.1.
- Платёжный провайдер и Telegram-бот требуют внешних ключей/настройки окружения (`.env`); на этапе тестов используются моки и mock-вебхук.
- Схемы шаблонов на MVP хранятся в коде (`/templates/*.ts`); модель данных готова к будущему переносу схем в БД без изменения рантайма.
